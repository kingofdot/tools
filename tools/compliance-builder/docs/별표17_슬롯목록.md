# 별표17 슬롯 매핑

별표7의 명명 규칙을 따라 별표17 답변에 `{{slot}}` 플레이스홀더를 매핑.

## 명명 규칙 (별표7 컨벤션)

- 차량: `{{<vehicleType>VehicleNo_<scope>}}` (예: `compactVehicleNo_Gen`, `closedVehicleNo_Haz`, `ctVehicleNo_Notif`)
- 시설: `{{<facilityName>Spec_<scope>}}` (예: `storageSpec_Gen`, `recyclingSpec_Haz`, `storageSpec_Notif`)
- 공통: `{{contactAddr}}`, `{{techStaff}}`, `{{parkingSpec}}`, `{{carWashSpec}}`, `{{labSpec}}` 등 scope 없음

`scope` 종류:
- `Gen` 생활폐기물
- `Ind` 사업장일반폐기물
- `Haz` 지정폐기물
- `Med` 의료폐기물
- `Notif` 폐기물처리 신고자 (별표17 신규 도입)

## 슬롯 목록 (별표17)

| 슬롯 | 사용 위치 | 의미 |
|---|---|---|
| `{{ctVehicleNo_Notif}}` | idx=2 (1.가.), idx=7 (2.다.) | 신고자 수집·운반차량 1대 이상 |
| `{{contactAddr}}` | idx=3 (1.나.) | 연락장소 또는 사무실 (별표7과 공유) |
| `{{storageSpec_Notif}}` | idx=5 (2.가.) | 신고자 보관시설(1일 처리능력의 1일분~30일분) |
| `{{recyclingSpec_Notif}}` | idx=6 (2.나.) | 신고자 재활용시설(선별·압축·감용·절단·사료화·퇴비화 중 해당 시설 1식 이상) |

## 비고 (idx 8~11)

비고 1~3은 예외/조건 설명이라 슬롯 매핑 없이 평문 유지.

## 신규 도입 슬롯 (Notif scope)

별표17에서 새로 추가된 신고자(Notif) scope 슬롯 4종:
- `ctVehicleNo_Notif`
- `storageSpec_Notif`
- `recyclingSpec_Notif`
- (재사용: `contactAddr`)

별표7과 동일하게 폼 매핑 시 사용자 입력 위젯 정의 필요.
