# 준수계획서 답변 작성 규칙

> 이 문서는 별표5 JSON의 `answer` 필드를 작성하거나 수정할 때 반드시 준수해야 하는 규칙이다.  
> 새로운 규칙이 확정되면 이 문서에 추가한다.  
> 수정 요청이 들어오면 이 문서에 위반되는 항목이 없는지 **먼저 확인 후 진행**한다.

---

## 1. 의무/허용 표현 구분 (핵심 원칙)

법령 문장 끝의 표현에 따라 답변 형식이 **완전히 다르다**.

### 1-1. 의무 표현 → `~하겠음.`

| 법령 원문 끝 표현 | 답변 형식 |
|---|---|
| `~하여야 한다` | `~하겠음.` |
| `~해야 한다` | `~하겠음.` |
| `~하여야 하며` | `~하겠음.` |
| `~하지 않아야 한다` | `~하지 않겠음.` |
| `~아니 된다` / `~아니하여야 한다` | `~하지 않겠음.` |

**예시**
```
법령: 사업장배출시설계 폐기물과 사업장비배출시설계 폐기물이 혼합되지 않도록 해야 한다.
답변: 사업장배출시설계 폐기물과 사업장비배출시설계 폐기물이 혼합되지 않도록 분리 관리하겠음.
```

---

### 1-2. 허용/임의 표현 → `~할 수 있음을 숙지하겠음.`

| 법령 원문 끝 표현 | 답변 형식 |
|---|---|
| `~할 수 있다` | `~할 수 있음을 숙지하겠음.` |
| `~처리할 수 있다` | `~처리할 수 있음을 숙지하겠음.` |
| `~수집·운반할 수 있다` | `~수집·운반할 수 있음을 숙지하겠음.` |
| `~인도할 수 있다` | `~인도할 수 있음을 숙지하겠음.` |

> **절대 금지**: 허용 표현에 `준수하겠음` 사용 불가.  
> `할 수 있다` 조항은 의무가 아닌 권한이므로, 이행 약속이 아닌 숙지 확인으로 답변한다.

**예시**
```
법령: 생활폐기물의 기준 및 방법으로 처리할 수 있다.
답변: 생활폐기물의 기준 및 방법으로 처리할 수 있음을 숙지하겠음.
```
```
법령: 동물의 사체는 동물장묘시설에서 처리할 수 있다.
답변: 동물의 사체는 동물장묘시설에서 처리할 수 있음을 숙지하겠음.
```

## 2. 번호 표기 규칙

### 2-1. 실제 마커 그대로 사용

법령 내 하위 항목을 답변에서 언급할 때는 **법령에 실제로 쓰인 마커**를 그대로 사용한다.

| 법령 마커 형식 | 답변에서 사용 | 금지 |
|---|---|---|
| `가)`, `나)`, `다)` | `(가)~(다)` | `①~③` |
| `(가)`, `(나)`, `(다)` | `(가)~(다)` | `①~③` |
| `1)`, `2)`, `3)` | `1)~3)` | `①~③` |
| `가.`, `나.`, `다.` | `가.~다.` | `①~③` |

> **동글번호(①②③) 사용 금지**: 법령에서 쓰지 않은 번호를 임의로 생성하면 독자가 혼동한다.

**예시**
```
❌ 잘못된 예: 아래 ①~③ 경우에는 밀폐형 덮개 설치차량으로 수집·운반하겠음
✅ 올바른 예: 아래 (가)~(다) 중 해당하는 경우에는 밀폐형 덮개 설치차량으로 수집·운반할 수 있음을 숙지하겠음.
```

### 2-2. 하위 항목을 나열하는 부모 항목 답변 형식

부모 항목이 `다음의 어느 하나에 해당하는 경우`처럼 OR 조건 목록을 소개할 때:

```
[부모 원칙] 원칙에도 불구하고, 아래 (가)~(다) 중 해당하는 경우에는 [예외 행위]를 [의무/허용에 따라] 하겠음.
```

## 3. 답변 문체 원칙

### 3-1. 법령 원문을 그대로 복붙하지 않는다

법령 원문은 길고 복잡하므로, **핵심 의무/권한만 추출해서 간결하게** 작성한다.

```
❌ 잘못된 예:
사업장비배출시설계 폐기물로서 생활폐기물과 성질과 상태가 비슷하여 생활폐기물의 기준
및 방법으로 처리할 수 있는 폐기물은 특별자치시, 특별자치도, 시·군·구의 조례에서
정하는 바에를 준수하겠음.

✅ 올바른 예:
사업장비배출시설계 폐기물로서 생활폐기물과 성질·상태가 비슷한 경우, 해당 지자체
조례에서 정하는 바에 따라 생활폐기물의 기준 및 방법으로 처리할 수 있음을 숙지하겠음.
```

### 3-2. 조사 오류 금지

자동 생성 시 자주 발생하는 조사 오류:

| 잘못된 표현 | 올바른 표현 |
|---|---|
| `배출를` | `배출을` |
| `바에를` | `바를` |
| `다목를` | `다목을` |
| `~다를 준수하겠음` | (전체 재작성 필요) |

> 특히 `~를 준수하겠음` 앞에 동사 원형이 붙는 경우 (`처리할 수 있다를 준수하겠음`) 는 반드시 전체 재작성.

### 3-3. 답변 끝 형식 통일

- 의무: `~겠음.` (마침표로 끝)
- 허용: `~숙지하겠음.` (마침표로 끝)
- 해당없음: `해당사항 없음` (마침표 없음)

## 4. 특수 케이스 처리

### 4-1. 단서 조항 (`다만, ~`)

법령에 단서가 있는 경우 답변에도 단서를 반영한다.

```
법령: ...탱크로리로 수집·운반하여야 한다.
      다만, 밀폐된 전용 수거용기에 담아 수집·운반하는 경우에는 ...할 수 있다.
답변: 탱크로리로 수집·운반하겠음.
      다만, 밀폐된 전용 수거용기에 담아 수집·운반하는 경우 적용 예외를 숙지하겠음.
```

### 4-2. 예외 허용 항목 (`~에도 불구하고, ~할 수 있다`)

`~에도 불구하고` 패턴은 원칙의 예외를 규정하므로, 허용 표현(`숙지하겠음`) 형식 사용.

```
법령: (1)에도 불구하고 ...밀폐형 덮개 설치차량으로 수집·운반할 수 있다.
답변: 원칙에도 불구하고, 아래 (가)~(다) 중 해당하는 경우에는
      밀폐형 덮개 설치차량으로 수집·운반할 수 있음을 숙지하겠음.
```

### 4-3. 해당사항 없음 (`_na` 항목)

wasteCode 또는 physicalState가 선택한 폐기물 유형과 맞지 않는 항목은 답변란에:

```
해당사항 없음
```

> 이 항목은 체크 불가, Word 출력 제외.

## 5. 태그 작성 시 확인사항 (answer와 연동)

| 태그 | 원칙 |
|---|---|
| `action: null` | 모든 처리행위에 공통 적용 (공통사항). answer는 의무/허용 표현에 따라 작성. |
| `wasteClass` | 가장 중요한 분류. 반드시 확인 후 태그. **D / GO / GN / L** 구분 (v0.3 이후 G → GO/GN 분리). |
| `physicalState: ["S"]` | 고상 전용. 다른 상태 선택 시 "해당사항 없음" 처리됨. |
| `physicalState: ["L"]` | 액상 전용. 마찬가지. |
| `noWord: true` | Word 출력 제외 (부칙·경과조치 등). answer 불필요. |
| `삭제` | text가 "삭제"로 시작하면 체크리스트에서 완전 제외. answer 불필요. |

### 5-1. wasteClass 세부 규칙 (v0.3)

| 값 | 의미 | 판단 근거 |
|---|---|---|
| `D` | 지정폐기물 | wasteCode `01-xx` ~ `11-xx` (의료폐기물 `10-xx` 포함) |
| `GO` | 사업장**배출시설계** 폐기물 | 법 제2조제3호 배출시설·영 제2조제1호~5호 시설에서 배출. 인허가 사업장. |
| `GN` | 사업장**비배출시설계** 폐기물 | 영 제2조제7호·9호 (연면적·폐기물량 기준 등) 사업장에서 배출. |
| `L` | 생활폐기물 | wasteCode `91-xx` 또는 가정배출. |

> **중요**: GO/GN은 wasteCode로는 구분되지 않는다(둘 다 `51-xx` 등 사업장일반폐기물 코드 사용).
> 배출시설 인허가 여부로만 구분되므로 **사용자 입력(facility permit)** 이 필요하다.

### 5-2. 🚨 섹션 엄격 분리 원칙 (2026-04-21 추가, **가장 중요한 원칙**)

**상위법령의 섹션(사업장 / 생활)은 절대 혼재시키면 안 된다.**

| 상위 섹션 | 허용 wasteClass | 허용 wasteCode 접두 |
|---|---|---|
| `1. 생활폐기물의 기준 및 방법` | `["L"]` 전용 | `91-xx` 전용 |
| `2. 음식물류 폐기물의 기준 및 방법` | `["L"]` 전용 | `91-02` 전용 (품목 = 음식물류) |
| `3. 사업장일반폐기물의 기준 및 방법` | `["GO"]`/`["GN"]`/`["GO","GN"]` | `51-xx` 전용 |
| `4. 지정폐기물의 기준 및 방법` | `["D"]` 전용 | `01-xx`~`12-xx` (+ 지정 편입 `51-08`, `51-09`, `51-22`, `51-30-05`) |

**규칙:**
1. **사업장 섹션 하위에 생활폐기물 코드/클래스 절대 금지** (`91-xx`, `L` 불가)
2. **생활/음식물류 섹션 하위에 사업장 코드/클래스 절대 금지** (`51-xx`, `GO`, `GN` 불가)
3. 지정폐기물 섹션에서만 예외적으로 `51-08`(소각재) 등 지정 편입 사업장코드 + `D` 조합 허용
4. 같은 품목(예: 음식물류, 오니)이 사업장·생활 양쪽에 나와도 **섹션에 해당하는 코드만** 써야 함
   - 사업장 음식물류 → `51-38`
   - 생활 음식물류 → `91-02`
   - 사업장 유기성 오니 → `51-01`
   - 지정 유기성 오니 → `01-02`

**검증:** `scripts/audit_b5_integrity.py` A-3 블록 (섹션 경계 기반 교차검증)

**2026-04-21 84건 교정 완료 근거:** `scripts/fix_b5_class_code.py` 참조 (인덱스별 명시 판정)

### 5-3. 별표5 재분류 규칙 (retag_wasteclass_go_gn.py 참조)

법령 text에 등장하는 용어에 따라 자동 결정:

| text에 등장 | 재태그 결과 |
|---|---|
| "사업장배출시설계"만 | `["GO"]` |
| "사업장비배출시설계"만 | `["GN"]` |
| 둘 다 등장 | `["GO","GN"]` |
| 둘 다 없음 | 부모 섹션 제목(가./나./...) 확인 후 동일 규칙 적용 |
| 여전히 불명 | 기본값 `["GO","GN"]` (사업장일반폐기물 전체) |

## 5-4. 🚨 별표5의3 예외 원칙 (2026-04-22 추가)

**별표5의3(재활용 기준)은 다른 별표와 달리 하위 세부 규칙까지 `answer`와 폐기물별 태그를 작성한다.**

이유:
- 별표5의3은 R-code별 재활용 유형이 폐기물 종류에 따라 매우 다르게 규정됨 (폐가전제품·폐합성수지·폐목재·폐촉매 등)
- 준수계획서 작성자가 자기 사업장의 폐기물 종류와 매칭되는 R-code·세부 규칙을 식별해야 하므로 폐기물별 태깅이 필수

규칙:
1. `depth 2` (R-x 유형 헤더): `rCode` 태그만 부여, `answer`는 상위 소개용
2. `depth 3` (R-x-y 세부 유형 헤더): `rCode` + 주요 `wasteCode` 태그 부여, `answer` 간단 작성
3. `depth 4` (폐기물별 규칙): 폐기물별 `wasteCode`/`wasteClass` 태그 부여, `answer` 상세 작성
4. `depth 5` (부속 세부 항목): `answer`는 짧게, 태그는 부모로부터 상속 (별도 부여 안 함)

예시:
```
"depth": 4, "text": "폐가전제품을 재활용하는 경우..."
  → "wasteCode": ["91-09"] (폐가전제품)
  → "answer": "폐가전제품 재활용 시 냉매물질 안전 회수, 모니터 유리 분리, 발포단열재 원료 재활용, 인쇄회로기판 별도 분리하겠음."
```

**이 예외는 별표5의3에만 적용된다.** 다른 별표(별표5, 별표8 등)는 기존 원칙(하위 답변 작성 안 함) 유지.

## 6. 수정 요청 전 체크리스트

수정 전 아래 항목을 먼저 확인한다:

- [ ] 해당 항목의 법령 원문 끝 표현이 `~하여야 한다`인가, `~할 수 있다`인가?
- [ ] 번호 표기가 실제 법령 마커와 일치하는가? (동글번호 없는가?)
- [ ] 조사가 정확한가? (`를/을`, `가/이` 등)
- [ ] 법령 원문을 그대로 복붙하지 않았는가?
- [ ] 단서 조항이 있다면 반영했는가?
- [ ] 허용 항목에 `준수하겠음`을 쓰지 않았는가?
- [ ] wasteClass가 `G`로 남아있지 않은가? (v0.3 이후 `GO`/`GN`으로 분리)
- [ ] wasteCode가 `wasteInformation.json`의 코드 체계와 일치하는가? (XX-XX 또는 XX-XX-XX)
- [ ] **상위 섹션(사업장 vs 생활)이 현재 wasteClass·wasteCode와 일치하는가?** (§5-2 섹션 엄격 분리 원칙)
