## 2. wasteVarMap.json 매핑 오류 (2026-04-21 해결)

사용자가 플래그한 `폐산 → G=51-01` 등 대대적인 매핑 오류.

**조치 (2026-04-21)**:
- [`scripts/validate_wastevar_map.py`](../scripts/validate_wastevar_map.py) 로 전수 검사 → 44건 문제 식별
- `data/wasteVarMap.json` 전면 재작성:
  - G → GO/GN 분리 (사업장일반 코드는 둘 다 동일)
  - 지정폐기물 매핑 수정: 폐산 `02-01`, 폐알칼리 `02-02`, 폐유 `06`, 폐촉매 `03-07`, 폐흡착제/폐흡수제 `03-08`, 폐석면 `07-01`, 폐농약 `01-03`, 폐유기용제 `04`
  - 사업장일반 세분화: 폐합성수지 `51-03-01`, 폐합성고무 `51-03-02`, 음식물류 `51-38`, 폐섬유류 `51-27`, 폐목재 `51-20`, 고철 `51-29-01`, 오니 `51-01/51-02`
  - 신규 변수 추가: 유기성/무기성오니류, 폐페인트, 폐락카, 폐의약품류, 연소잔재물, 폐석재류, 폐콘크리트류, 안정화처리물, 동식물성잔재물, 폐냉매물질 등
  - 제거: 폐용기류 (근거 코드 없음)
- 재검증 결과: **128건 매핑 중 문제 0건**

## 3. phaseState vs physicalState 혼재 (2026-04-21 해결)

별표5 일부 항목이 `"phaseState": "liquid"` / `"physicalState": ["L"]` 두 키를 모두 가졌음.

**결정**: `physicalState` 를 정본으로 삼고, `phaseState` 는 전면 제거.

**조치 (2026-04-21)**: [`scripts/unify_physicalstate.py`](../scripts/unify_physicalstate.py) 실행으로
- 35건: `phaseState: "solid"|"liquid"` → `physicalState: ["S"]|["L"]` 로 마이그레이션
- 318건: `phaseState: null` 단순 제거
- 13건: `physicalState` 이미 존재 → `phaseState` 만 제거
`phaseState` 키 잔존 0건 확인.

## 4. wasteClass "G" 전면 재분류 (2026-04-21 완료)

별표5 내 `"wasteClass": ["G"]` 111건 → `["GO"]` / `["GN"]` / `["GO","GN"]` 로 재분류 완료.
스크립트: [`scripts/retag_wasteclass_go_gn.py`](../scripts/retag_wasteclass_go_gn.py)

재분류 기준 요약:
- text "사업장배출시설계"만 등장 → `["GO"]`
- text "사업장비배출시설계"만 등장 → `["GN"]`
- 둘 다 / 둘 다 없음 → `["GO","GN"]`

## 6. wasteClass ↔ wasteCode 교차불일치 대규모 정리 (2026-04-21 해결)

### 6-1. 발견

`scripts/audit_b5_integrity.py` A-3 스캔 결과 **90건의 wasteClass × wasteCode 불일치** 검출.
주요 패턴:
- 사업장일반 섹션 하위에 지정폐기물 코드(`01-02`) 태그 — 22건
- 지정폐기물 섹션 하위에 사업장일반 코드(`51-05`, `51-08` 등) 태그 — 50건 이상
- 음식물류·생활 섹션 하위에 `GO/GN` 태그 (L이어야 함) — 25건 이상
- 그 외 수집·운반증 섹션에 `D` 오태깅

### 6-2. 원칙 수립 — 🚨 섹션 엄격 분리 (답변작성규칙 §5-2)

상위법령에서 구분된 섹션(사업장 / 생활)은 **하위에 절대 혼재 불가**.
지정폐기물 섹션만 예외적으로 지정 편입 사업장코드(`51-08`, `51-09`, `51-22`, `51-30-05`) + `D` 허용.

### 6-3. 조치

1. `scripts/fix_b5_class_code.py` — 84건 인덱스별 수동 판정 반영 (자동 일괄 규칙 금지, 각 항목 본문·부모 섹션·별표4 세부분류코드 대조 후 명시 판정)
2. `scripts/audit_b5_integrity.py` 에 섹션 경계 기반 교차검증(`parent_section_kind`) 추가
3. 재스캔 결과: **A-3 불일치 0건**, 천연방사성 예외 24건만 참고로 남음
