## 1. wasteInformation.json 누락 코드 (2026-04-21 발견)

별표5에서 사용되지만 `data/wasteInformation.json` 에 없는 wasteCode:

| 코드 | 명칭 | 별표5 출현 | 비고 |
|---|---|---|---|
| `12-00-00` | 천연방사성제품폐기물 | 11회 | 법 별표1 제12호. 방사성 안전관리 대상. |
| `91-20-00` | 천연방사성제품생활폐기물 | 2회 | 생활폐기물 섹션 11)번에 등장. |

**영향**  
자동 검증 및 UI 드롭다운에서 해당 코드를 선택할 수 없음 → 천연방사성 관련 항목이 필터링되지 않음.

**조치 필요**  
`wasteInformation.json` 에 위 2개 코드 추가. `recyclingCodeNone`·`recyclingCodeCorrespond` 는 "재활용 불가" 로 보는 것이 안전.

## 2. wasteVarMap.json 매핑 오류 가능성 (2026-04-21 플래그)

사용자가 직접 지적:

- `폐산` → G=51-01? (잘못됨. 51-01은 "유기성오니류". 폐산은 지정폐기물 계열)

별표5 재빌드 전에 `wasteVarMap.json` 전체 재검증 필요.

## 3. phaseState vs physicalState 혼재 (2026-04-21 확인)

별표5 일부 항목이 `"phaseState": "liquid"` / `"physicalState": ["L"]` 두 키를 모두 가짐.

**결정**: `physicalState` 를 정본으로 삼고, 상황코드·build.js·필터링 로직은 `physicalState` 만 참조.
`phaseState` 는 legacy 로 유지하되 신규 태그는 사용 금지.

## 4. wasteClass "G" 전면 재분류 (2026-04-21 완료)

별표5 내 `"wasteClass": ["G"]` 111건 → `["GO"]` / `["GN"]` / `["GO","GN"]` 로 재분류 완료.
스크립트: [`scripts/retag_wasteclass_go_gn.py`](../scripts/retag_wasteclass_go_gn.py)

재분류 기준 요약:
- text "사업장배출시설계"만 등장 → `["GO"]`
- text "사업장비배출시설계"만 등장 → `["GN"]`
- 둘 다 / 둘 다 없음 → `["GO","GN"]`
