# Word 출력 설계 원칙

## 1. noWord 필드 — 출력 제외 항목

### 용도
별표 JSON 항목 중 Word 문서에 인쇄할 필요 없는 항목에 `"noWord": true` 필드를 추가한다.

### 대상
- 부칙·경과조치 (시행일 적용 지역, 날짜)
- 별표 제목이 파싱 오류로 본문에 혼입된 항목

### 사용법
```json
{
  "depth": 0,
  "type": "number",
  "marker": "1.",
  "text": "서울특별시, 인천광역시 및 경기도 지역: 2026년 1월 1일",
  "tags": null,
  "noWord": true
}
```

### 적용 현황 (별표5)
- `idx=1`: 서울·인천·경기 시행일 (2026. 1. 1.)
- `idx=2`: 제1호 외 지역 시행일 (2030. 1. 1.)

### 코드 처리 위치
`exportToWord()` 함수 내:
```javascript
const B5_WORD_FILTERED = B5_ALL_FOR_WORD.filter(i => !i.noWord);
```

## 2. 최상위 섹션 전체 미해당 처리

### 문제
사용자가 "사업장일반폐기물"만 선택한 경우, 생활폐기물 섹션 전체가 해당 없다.
기존: 생활폐기물 내 서브헤더(가. 공통사항, 나. 수집·운반 등)마다 "해당없음" 행이 추가됨.
현재: 최상위 섹션 헤더 1행에 "해당없음" 표기 후 내부 전부 생략.

### 동작 기준
- **최상위 섹션(depth=0)** 아래 선택된 항목이 하나도 없으면 → 헤더 1행만 출력, 내부 서브헤더·데이터 전부 생략
- 서브헤더(depth≥1) 아래 선택 항목이 없으면 → 그 서브헤더 1행 + "해당없음" 우측 출력, 내부 항목 생략

### 적용 대상
모든 검토사항 별표에 동일 원칙 적용. 새 별표를 추가할 때도 동일 로직 재사용.

## 3. "해당없음" 위치 — 우측 열 같은 행

### 기존 방식
```
행1: [섹션헤더 텍스트]  |  (빈 칸)
행2: (빈 칸)            |  해당하지 않으므로 기재하지 않음
```

### 현재 방식
```
행1: [섹션헤더 텍스트]  |  해당하지 않으므로 기재하지 않음
```
헤더와 같은 행 우측에 회색(#888888)으로 출력.

### 예외
항목이 하나 이상 선택된 섹션 → 기존과 동일하게 헤더만 출력, 우측 빈 칸.

## 4. 다른 별표 파일 확장 시 체크리스트

새 별표 파일을 검토사항에 추가할 때:
1. 부칙·경과조치 항목에 `"noWord": true` 추가
2. 파싱 오류로 별표 제목이 본문에 혼입된 항목 수정 또는 `"noWord": true` 처리
3. 최상위 섹션 구조(depth=0 헤더)가 폐기물 분류(생활/사업장/지정/의료)와 일치하는지 확인
4. exportToWord 함수에서 `B5_ALL_FOR_WORD` 대신 해당 별표 배열 이름 사용
