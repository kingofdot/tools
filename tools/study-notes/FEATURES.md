# Study Notes — 기능 요약

공부 내용을 필기하고 모바일·데스크탑에서 동일하게 볼 수 있도록 만든 개인용 노트 앱.
**서버 없음, 정적 HTML/CSS/JS.** 저장은 브라우저 localStorage + GitHub(백업 + 크로스 디바이스 동기화).

---

## 0. 전체 흐름 한 눈에

```
┌─ 상단바 ──────────────────────────────────────────────┐
│ Study Notes   [🔍 검색]           ● ⇅ ⌨ ⚙             │
├─ 과목 탭 ───────────────────────────────────────────┤
│ 1 민법  2 형법  3 헌법  +                              │
├─ 좌측 목록 ─────┬─ 우측 편집 ─────────────────────────┤
│                  │ [과목] [주제]        [두문자]        │
│ + 새 노트        │ [📖 보기] [✏ 편집]            🗑   │
│ ⋮⋮ 매매의 효력  │──────────────────────────────────│
│ ⋮⋮ 채권자취소   │ (렌더된 본문 또는 textarea)         │
│                  │                                    │
└─────────────────┴──────────────────────────────────┘
```

## 1. 데이터 모델

### 노트 한 건
```ts
{
  id:        string    // ISO timestamp 기반 고유값
  subject:   string    // 과목 (예: "민법")
  topic:     string    // 주제 (예: "매매의 효력")
  mnemonic:  string    // 두문자 (예: "재대과")
  body:      string    // 들여쓰기로 구조화된 본문 텍스트
  order:     number    // 같은 과목 내 정렬 순서
  updatedAt: string    // ISO 8601 UTC
}
```

### 전체 저장 포맷 (localStorage + GitHub 단일 JSON)
```json
{
  "notes": [ ...위 구조... ],
  "savedAt": "2026-04-24 12:34:56"
}
```

한 파일에 모든 노트를 담습니다 — **개별 노트당 파일을 만들지 않음**. 저장소 분산 없어 sha 충돌도 파일 한 개 단위로 단순.

## 2. 입력 규칙 (공부 노트 특화)

### 2.1 들여쓰기 = 깊이
- **Tab** → 깊이 +1
- **Shift+Tab** → 깊이 -1
- **Enter** → 새 줄 (직전 들여쓰기 유지)
- 공백 4개도 Tab 1개로 인식

### 2.2 깊이별 자동 번호 마커
| 깊이 | 마커 | 예 |
|---|---|---|
| 0 | 로마자 대문자 | `I.` `II.` |
| 1 | 아라비아 | `1.` `2.` |
| 2 | 반괄호 | `1)` `2)` |
| 3 | 원형 숫자 | `①` `②` |
| 4+ | 한글 | `가.` `나.` |

내부 트래킹 번호는 `1.2.3` 같은 dotted 체계지만 화면에는 위 마커만 찍힙니다.

### 2.3 선두 숫자 자동 무시 (복붙 지원)
외부에서 가져온 텍스트가 이미 번호를 가지고 있으면 파서가 **자동 제거 + 점 개수로 깊이 추정**.
```
1.1. 민원의 접수              ← 2세그먼트 → depth 1
1.1.1. 행정기관의 장은...      ← 3세그먼트 → depth 2
1.2.1.2.3.4. 접수증을 갈음...  ← 6세그먼트 → depth 5
```
탭이 함께 있으면 탭 우선.

### 2.4 백틱 본문 접두
줄 맨 앞에 `` ` ``(백틱)을 붙이면 **번호 없는 본문**으로 렌더. 크기는 depth 3 본문.
"로마자 큰 제목 바로 아래 들어가는 보조 설명"용.
```
I. 매매의 효력
`매매계약에서 발생하는 권리·의무의 내용.
    1. 재산권 이전
```

### 2.5 자동 강조 (inline)
| 패턴 | 예 | 표시 |
|---|---|---|
| `(제N조 제N항)` `(제N조)` 등 | `(제568조 제1항)` | 파란 뱃지 |
| `NNN다NNNN` / `NN도NN` / `헌마NN` 등 판례번호 | `2004다8210` | 보라 뱃지 |

둘 다 클릭하면 클립보드 복사.

## 3. 조직화

### 3.1 과목 탭 (최상단)
- 노트의 `subject`를 자동 수집해 과목별 탭으로 그룹화
- 활성 탭의 과목 노트만 좌측 목록에 표시
- 탭 앞에 `1 2 3 ...` 인덱스 → **Alt+숫자**로 바로 전환
- `+` 버튼: 새 과목 생성 (빈 노트 한 개 자동 추가)
- 메타 행의 **과목** 입력 필드로도 즉시 변경 가능 (없는 과목 입력 시 탭 자동 생성)

### 3.2 좌측 노트 목록
- 같은 과목 안의 노트 리스트 (세로)
- `⋮⋮` 드래그 핸들로 순서 변경 (HTML5 drag & drop)
- 각 항목에 주제·두문자 뱃지·수정일 표시
- 검색어 입력 시 실시간 필터

## 4. 보기 / 편집 모드

**상단 메타 행의 토글** 또는 **Ctrl+E**로 전환.

### 📖 보기 (study mode)
- 책처럼 렌더된 본문 한 덩어리
- 상단에 주제 + 두문자 뱃지 헤더 (선택적)
- 종이 톤 배경 + 세리프 폰트 (Noto Serif KR)
- 깊이별 타이포그래피 위계 (큰 제목 밑줄 등)

### ✏ 편집 (edit mode)
- 좌측: 탭 들여쓰기 가능한 textarea
- 우측: 실시간 렌더 미리보기 (같은 내용, 헤더 제외)

## 5. 검색

상단바 검색창 — **과목 · 주제 · 두문자 · 본문** 모두 대상.
- 좌측 노트 목록 즉시 필터
- 현재 노트에서도 매치 텍스트에 노란 하이라이트 (`mark.search-hit`)
- 단축키 **Ctrl+K** 로 포커스

## 6. 동기화 (GitHub)

### 6.1 방식
- **저장소**: `kingofdot/tools` (이미 존재하는 repo 사용 — 별도 저장소 생성 불필요)
- **경로**: `tools/study-notes/notes-data.json`
- **인증**: fine-grained PAT (Contents: Read & Write)
- 단일 파일 GET/PUT 로 전체 노트 교환 (prisma-architect 방식)

### 6.2 타이밍
- **로컬 저장**: 모든 입력 → localStorage 즉시 저장 (오프라인 OK)
- **자동 푸시**: 편집 완료 후 5초 디바운스 → 전체 notes-data.json PUT
- **수동 ⇅**: pull → 로컬 병합 → push
  - 병합 규칙: `updatedAt`이 더 새로운 쪽으로 (newer-wins)

### 6.3 상태 표시 (⬤ 점)
- 초록: 동기화됨
- 노랑: 저장 대기(dirty)
- 빨강: 오류

## 7. PWA (모바일 지원)

- `manifest.json` + `sw.js` Service Worker
- **홈 화면에 추가**하면 풀스크린 앱처럼 동작
- 정적 자산은 캐시되어 **오프라인에서도 열람·편집 가능** (동기화만 네트워크 복귀 시)
- 캐시 버전 `study-notes-vN` — 코드 변경 시 N을 올리면 사용자가 새로고침만 해도 최신판 적용

## 9. 코드 구조

```
study-notes/
  index.html           레이아웃 (상단바 · 과목 탭 · 목록 · 편집 · 모달들)
  style.css            디자인 토큰 + 전체 스타일 (다크 모드 자동)
  manifest.json        PWA 메타
  sw.js                Service Worker (정적 자산 캐시)
  js/
    state.js           전역 상태 변수 (notes, settings, viewMode ...)
    storage.js         localStorage CRUD, 노트 템플릿, 드래그 정렬
    parser.js          들여쓰기 텍스트 → 트리 (선두 번호 무시, 백틱 접두)
    numbering.js       depth → 마커 문자열 (로마자/아라비아/반괄호/원형/한글)
    highlight.js       법조문·판례 정규식 매칭 + 검색 하이라이트
    render.js          트리 → HTML (보기/편집 미리보기 공용)
    editor.js          Tab/Enter 키 처리, 모드 토글, 저장 디바운스
    list.js            과목 탭 + 노트 목록 + 드래그 정렬 + 단축키 스위칭
    search.js          검색 필터
    github.js          Contents API GET/PUT, base64 인코딩
    sync.js            pull / push / fullSync
    init.js            부팅, 전역 이벤트 바인딩, 전역 단축키, 모달
```

## 12. 디자인 방향 힌트

"공부하는 느낌"이라는 목표에 맞춰 잡아놓은 축:
- **종이 + 만년필**: 크림색 배경 + 잉크 그린 액센트
- **세리프 본문**: 긴 글 가독성 우선. 기술적 tone 줄임.
- **번호 체계 자동화**: 사용자는 들여쓰기에만 집중, 번호 매김은 앱이.
- **법조문/판례 하이라이트**: 법학·행정학 노트에 특화된 시그널.

재디자인 할 때 고려할 점:
- 좌측 목록을 완전히 감추는 "zen mode"
- 보기 모드에 "인쇄용 단색 테마"
- 모바일 세로 화면에서 탭을 하단 Bottom Nav로 내리기
- 큰 제목 폰트 옵션 (명조 ↔ 고딕 스위치)
